#################### Clone git repository on local system ###################

1. Go to GitHub server and open newly created or old repository.
2. click on code button with green color.
3. choose HTTPS option and copy the repository URL.
   e.g. https://github.com/Pathmakp26/AutomationPractice.git

4. Open git Terminal inside GitHubRepos folder.
5. Run git clone command 
   ->  git clone https://github.com/Pathmakp26/AutomationPractice.git
   ->  Enter

6. Once the clone is successful use command dir/ls to mare sure repository folder created successfully on local system.

7. Manually from explorer verify that repository folder should be available inside the GitRepos folder.

############## Configure User Credentials on git client #########
1. Go to git terminal 
2. run command to open config file
   - git config --global --edit

3.  It will open Linux editor, we have to provide GitHub username and email.
4.  Initiate a insert mode by pressing i key on keyboard.
5.  Use arrow keys and remove # to un-comments the username and email lines.
6.  Update the correct git username and email
7.  Close the with below steps.
    -  press ESC key :  it will disabled insert mode
    -  type :wq and press enter :  It will save file and close the config file.
8. User configuration is successfully completed.

######### Add File to git repository ##############
1. Create a text file in repository folder and add some content.
2. Open git terminal inside repository folder.
3. Run git status command
   - git status
   - It will show new added file in untracked file section in red color.
4. Run git add command
   - git add <filename>
   - It will add file in git tracking.
5. Again run git status command
   - git status
   -  it will show files in green color.

6. Initiate commit operation
   - git commit -m "Add file to repo"
   - It will initiate commit operation.
  
7. Git push file to repo.
   - git push
   - If we are doing it first time, then it will launch a popup, that we have to choose "open in browser" option.
   -  It will launch git authentication screen
   - click on git authorize eco-system.
   - Push operation will successful.

8. Check file should be available on git server in dedicated repository.
